SRE Field Guide

The Emergency Manual.

Not a tutorial. Not a guide. Just the commands — you figure out which one to use.

🌐Networking
ss -tlnp
Show listening TCP sockets with process names
ip addr show
List all network interfaces and IPs
dig +short example.com
Quick DNS resolution check
netstat -rn
Show routing table
curl -Iv https://host
Verbose HTTP headers + TLS info
📋Log Analysis
journalctl -xe
System journal, jump to errors
journalctl -u nginx -f
Follow service logs in real time
tail -f /var/log/messages
Live tail of system messages
grep -i error /var/log/syslog
Filter errors from syslog
dmesg | tail -20
Kernel ring buffer — hardware issues
💾Disk Recovery
df -h
Human-readable disk usage per filesystem
du -sh * | sort -h
Size of each dir, sorted smallest → largest
find / -size +100M -type f
Find files larger than 100MB
lsof | grep deleted
Deleted files still held open by processes
> /var/log/app.log
Truncate a log without deleting it (safe)
⚙️Processes
ps aux --sort=-%cpu
All processes, sorted by CPU usage
top -b -n1 | head -20
One-shot top output, top 20 lines
strace -p <pid>
Trace system calls of a running process
kill -9 <pid>
Force kill — last resort
lsof -p <pid>
Files/sockets open by a process
🔒Security / SELinux
ls -Z /path
Show SELinux security context of files
ausearch -m avc -ts recent
Recent SELinux denials from audit log
restorecon -Rv /path
Restore default SELinux file context
iptables -L -n -v
List all firewall rules with packet counts
getenforce
Check if SELinux is enforcing or permissive
🛠️Services / systemd
systemctl status <svc>
Status + last journal lines for a service
systemctl list-units --failed
All failed units at a glance
systemctl restart <svc>
Restart a service gracefully
systemd-analyze blame
Boot time per service — find slow starters
journalctl --since "10 min ago"
Recent journal entries only
